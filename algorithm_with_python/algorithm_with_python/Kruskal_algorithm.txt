신장트리란? : 하나의 그래프가 있을때 모든 노드를 포함하면서, 사이클이 존재하지 않는 부분 그래프

11-22
l  l
33-44
이런 그래프가 있을때, 여러개의 신장트리를 가질수있는데, 예시로
11-22
   ㅣ
33-44
이게 있고, 안되는 예시는
11-22        11---22
ㅣ ㅣ        ㅣ
33-44        33  44
사이클 존재   연결안된 노드존재 등이 있다

이러한 신장트리를 최소비용으로 뽑는 알고리즘을 최소신장트리 알고리즘이라 하는데,
이의 종류 중 하나로 그리디알고리즘인 크루스칼 알고리즘이 있다

알고리즘은 간단한데
1. 간선의 비용을 오름차순으로 정렬한다
2. 간선을 하나씩 확인하며 지금까지의 간선과 사이클이 형성되는지 확인,
  -사이클이 형성되면 포함시키지 않고
  -사이클이 형성되지 않으면 포함시킨다
3. 모든 간선에 대해 2번 반복

위에서 2번에, 사이클을 찾는부분에서 사이클이 형성되는지 확인하는 부분은 
서로소 집합 알고리즘(find_union_algorithm)의 find를 이용해 루트노드값이 다른지 확인하는거고
신장트리에 포함시킬때는 union을 하고, 트리에 포함한다.
신장트리에 포함시키지 않을때는 union을 실행하지 않는다

그러면 코드는 쉽게 작성할수 있다.
